Getting Started with okBATHTUB

Overview

okBATHTUB implements steady-state empirical reservoir eutrophication modelling based on Walker’s (1996) BATHTUB framework, extended with Oklahoma-specific regression coefficients calibrated from OWRB Lake Monitoring Program (LMP) data.

The package predicts:

  • In-lake total phosphorus (TP) from tributary loading and hydraulic retention
  • Chlorophyll-a from in-lake TP via log-log regression
  • Secchi depth from chlorophyll-a via log-log regression
  • Carlson Trophic State Indices (TSI) from all three

It is designed to complement watershed loading models such as OK-HAWQS/SWAT by translating nutrient loads into in-lake water quality predictions.

Installation

# Install from GitHub (until CRAN submission)
# install.packages("remotes")
remotes::install_github("0011235813/okBATHTUB")
library(okBATHTUB)

The Core Pipeline

The standard single-segment workflow chains five functions with the base pipe operator:

ok_load() |> ok_hydraulics() |> ok_retention() |> ok_inlake() |> ok_tsi()

Every function returns an okBATHTUB S3 object. Data accumulates through the pipeline so the final result contains everything from every step.

Step 1 — Assemble inflow loads

ok_load() is the pipeline entry point. Supply annual inflow volume, flow-weighted mean TP concentration, and optionally TN:

result <- ok_load(
  inflow_m3yr   = 45e6,    # 45 million m³/yr (~1.43 m³/s mean flow)
  tp_inflow_ugl = 120,     # 120 µg/L flow-weighted mean TP
  tn_inflow_ugl = 1800,    # 1800 µg/L flow-weighted mean TN
  segment_label = "lacustrine"
)

print(result)
#> -- okBATHTUB Result --
#>   Pipeline step : load
#>   Segment       : lacustrine
#>   Coefficients  : walker
#> 
#>   inflow_m3yr    : 4.5e+07
#>   tp_inflow_ugl  : 120
#>   tp_load_kgyr   : 5400
#>   tn_inflow_ugl  : 1800
#>   tn_load_kgyr   : 8.1e+04

Step 2 — Reservoir hydraulics

ok_hydraulics() computes hydraulic residence time (τ) and areal water load (q_s) from reservoir morphometry:

result <- result |>
  ok_hydraulics(
    surface_area_ha = 890,   # normal pool surface area
    mean_depth_m    = 4.2    # mean depth at normal pool
  )

cat("Hydraulic residence time:", round(result$data$hydraulic_residence_time_yr, 2), "yr\n")
#> Hydraulic residence time: 0.83 yr
cat("Areal water load (qs):   ", round(result$data$areal_water_load_myr, 2), "m/yr\n")
#> Areal water load (qs):    5.06 m/yr

Step 3 — Nutrient retention

ok_retention() estimates the fraction of incoming TP and TN retained in the reservoir. The default uses the Larsen-Mercier (1976) formulation from Walker (1996):

\[R_{TP} = \frac{1}{1 + 1/\sqrt{\tau}}\]

result <- result |> ok_retention()

cat("TP retention coefficient:", round(result$data$tp_retention_coeff, 3), "\n")
#> TP retention coefficient: 0.477
cat("TN retention coefficient:", round(result$data$tn_retention_coeff, 3), "\n")
#> TN retention coefficient: 0.664

Step 4 — In-lake predictions

ok_inlake() applies the mass balance and regression equations:

result <- result |> ok_inlake()

cat("In-lake TP:    ", round(result$data$tp_inlake_ugl, 1), "µg/L\n")
#> In-lake TP:     62.8 µg/L
cat("Chlorophyll-a: ", round(result$data$chla_ugl, 2),    "µg/L\n")
#> Chlorophyll-a:  29.45 µg/L
cat("Secchi depth:  ", round(result$data$secchi_m, 2),    "m\n")
#> Secchi depth:   0.83 m

Step 5 — Trophic State Index

ok_tsi() computes Carlson (1977) TSI from all predicted variables:

result <- result |> ok_tsi()

Using the Oklahoma Reservoir Lookup

The ok_reservoirs dataset contains morphometry for all OWRB LMP reservoirs. Use ok_reservoir() instead of looking up surface area and mean depth manually:

# Look up Arcadia Lake from the built-in dataset
res <- ok_reservoir("Arcadia")
if (nrow(res) == 0) {
  # Fallback: use known Arcadia Lake morphometry if not in local dataset
  res <- data.frame(
    lake_name       = "Arcadia Lake",
    surface_area_ha = 890,
    mean_depth_m    = 4.2,
    eco_l3_name     = "Cross Timbers",
    data_quality    = "A",
    notes           = "Hardcoded fallback"
  )
}
cat("Surface area:", res$surface_area_ha, "ha\n")
#> Surface area: 890 ha
cat("Mean depth:  ", res$mean_depth_m, "m\n")
#> Mean depth:   4.2 m
# Use known Arcadia Lake morphometry for a reproducible example
result <- ok_load(
    inflow_m3yr   = 45e6,
    tp_inflow_ugl = 120,
    tn_inflow_ugl = 1800
  ) |>
  ok_hydraulics(
    surface_area_ha = 890,
    mean_depth_m    = 4.2
  ) |>
  ok_retention() |>
  ok_inlake()    |>
  ok_tsi()

cat("Trophic state:", result$data$trophic_state, "\n")
#> Trophic state: Eutrophic
cat("Mean TSI:     ", round(result$data$tsi_mean, 1), "\n")
#> Mean TSI:      63.4

Multiple Tributaries

When more than one tributary contributes to the reservoir, use ok_load_multi() to compute flow-weighted mean concentrations automatically:

tributaries <- data.frame(
  inflow_m3yr   = c(30e6, 15e6),
  tp_inflow_ugl = c(100,  160),
  tn_inflow_ugl = c(1500, 2400)
)

result_multi <- ok_load_multi(tributaries) |>
  ok_hydraulics(surface_area_ha = 890, mean_depth_m = 4.2) |>
  ok_retention() |>
  ok_inlake()    |>
  ok_tsi()

# Flow-weighted mean TP: (100*30e6 + 160*15e6) / 45e6 = 120 µg/L
cat("FW mean TP inflow:", round(result_multi$data$tp_inflow_ugl, 1), "µg/L\n")
#> FW mean TP inflow: 120 µg/L

Computing TSI from Observed Data

ok_tsi_observed() computes Carlson TSI directly from monitoring data without running the full pipeline — useful for checking observed trophic state against modelled predictions:

obs <- ok_tsi_observed(
  tp_ugl    = 85,
  chla_ugl  = 22,
  secchi_m  = 0.8
)

cat("TSI(TP):    ", round(obs$tsi_tp,     1), "\n")
#> TSI(TP):     68.2
cat("TSI(Chl-a): ", round(obs$tsi_chla,   1), "\n")
#> TSI(Chl-a):  60.9
cat("TSI(Secchi):", round(obs$tsi_secchi, 1), "\n")
#> TSI(Secchi): 63.2
cat("Mean TSI:   ", round(obs$tsi_mean,   1), "\n")
#> Mean TSI:    64.1
cat("Class:      ", obs$trophic_state,         "\n")
#> Class:       Eutrophic

Coefficient Sets

All pipeline functions accept a coefficients argument:

  • "walker" (default) — Walker (1996) nationally-derived coefficients
  • "oklahoma" — Oklahoma LMP-calibrated coefficients (Chl-a and Secchi regressions); Walker retention defaults

When using Oklahoma coefficients, supply the EPA Level III ecoregion for best results:

# Walker coefficients (national defaults)
r_walker <- ok_load(
    inflow_m3yr   = 45e6,
    tp_inflow_ugl = 120,
    coefficients  = "walker"
  ) |>
  ok_hydraulics(surface_area_ha = 890, mean_depth_m = 4.2) |>
  ok_retention() |>
  ok_inlake()    |>
  ok_tsi()

# Oklahoma ecoregion-specific coefficients
r_oklahoma <- ok_load(
    inflow_m3yr   = 45e6,
    tp_inflow_ugl = 120,
    coefficients  = "oklahoma",
    ecoregion     = "Cross Timbers"
  ) |>
  ok_hydraulics(surface_area_ha = 890, mean_depth_m = 4.2) |>
  ok_retention() |>
  ok_inlake()    |>
  ok_tsi()

cat("Chlorophyll-a (Walker):   ", round(r_walker$data$chla_ugl, 2),   "µg/L\n")
#> Chlorophyll-a (Walker):    29.45 µg/L
cat("Chlorophyll-a (Oklahoma): ", round(r_oklahoma$data$chla_ugl, 2), "µg/L\n\n")
#> Chlorophyll-a (Oklahoma):  24.65 µg/L
cat("Mean TSI (Walker):        ", round(r_walker$data$tsi_mean, 1),   "\n")
#> Mean TSI (Walker):         63.4
cat("Mean TSI (Oklahoma):      ", round(r_oklahoma$data$tsi_mean, 1), "\n")
#> Mean TSI (Oklahoma):       65.2

Oklahoma reservoirs typically show shallower Chl-a vs TP slopes than Walker’s national coefficients predict, because turbid, flood-dominated systems show a more muted algal response per unit phosphorus.
